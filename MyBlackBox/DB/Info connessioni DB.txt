INFO SQL SERVER: UTENTI, LOGIN,SCHEMA
	1) premessa: se instalo il db da situazione vergine devo seguire la strada dell'autenticazione windows dato che altrimenti devo agire su REGISTRY con regedit
	 per permettere l'autenticazione in due modalità: windows e sql serverv autentication
		1)The Authentication mode for MS-SQL Server can be changed after installation without installing the SQL Management Studio.

			a.Open the Window Registry Editor (Start --> Run --> regedit.exe).
			b.Go to "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Microsoft SQL Server\<INSTANCE NAME>\MSSQLServer where <INSTANCE NAME? is the name of the MS-SQL instance.
			c.Locate the field "LoginMode".  Change it's value to "2" to put MS-SQL into Mixed Mode Authentication.
			d.Restart the MS-SQL service.
			
			NB: il valore iniiale della chiave nel registry è 1

	2) se volesi utilizzare l'autenticazione SQL Server (SQLServer autentication anzichè Windows nt..) nello script potrei agire in questo modo:
		a. aggiungo il login: 
			CREATE LOGIN [mbb] WITH PASSWORD = N'******' ,
			  DEFAULT_DATABASE = [master],
			  DEFAULT_LANGUAGE = [English],
			  CHECK_POLICY = OFF,
			  CHECK_EXPIRATION = OFF
			GO
			
		b. aggiungo lo user e lo schema
			CREATE USER [mbb]
			  FOR LOGIN [mbb]
			  WITH DEFAULT_SCHEMA = [mbb]
			GO

			CREATE SCHEMA [mbb]
			  AUTHORIZATION [mbb]
			GO
			
			.in questo modo creo un utente che fa riferimento all'utente di login con lo stesso nome (per semplicità) mbb
			.autorizzo l'utente mbb (che deriva dal login mbb) ad utilizzare lo schema mbb
			
		c. a questo punto posso creare gli oggetti del db, che fanno riferimento al nuovo schema
			es:
			CREATE TABLE [mbb].[note] (
				[ID] int NOT NULL,
				..
				
		d. la connection string per sql server ora comprenderà anche
			dbUser= mbb, dbpwd=mbbcore (su file ini)
			che si traduce in user id=mbb; password=mbbcore
			
	3) in EMS ricorda di registrare il database con windows autentication o sql server authentication
		.in base alla modalità di accesso/registrazione, sarà possibile visualizzare gli oggetti del db
			a. se l'accesso è con sql server auth. e il db è stato generato con utente e pwd => vedo gli oggetti di mbb
			   altrimenti se entro con dbo (win auth.) non vedo niente
			b. se entro con win auth. (utente di default dbo) e genero il db con schema dbo, nello schema dbo vedrò le tabelle create da script

This problem occurs if the user tries to log in with credentials that cannot be validated. This problem can occur in the following scenarios:

Scenario 1: The login may be a SQL Server login but the server only accepts Windows Authentication.
Scenario 2: You are trying to connect by using SQL Server Authentication but the login used does not exist on SQL Server.
Scenario 3: The login may use Windows Authentication but the login is an unrecognized Windows principal. An unrecognized Windows principal means that Windows can't verify the login. This might be because the Windows login is from an untrusted domain.

To resolve this problem, follow the steps that apply to your scenario. Scenario 1: Configure SQL Server in Mixed Authentication Mode.

SQL Server 2012, SQL Server 2008, and SQL Server 2005 Open SQL Server Management Studio. To do this, click Start, click All Programs, click Microsoft SQL Server 200x (where x is the version of SQL), and then click SQL Server Management Studio. Right-click the server, and then click Properties. See image. On the Security page, under Server authentication, click the SQL Server and Windows Authentication mode option button, and then click OK.
